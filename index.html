Introduction

Welcome to our Inventory Management System documentation. Here, you'll find all the essential information to efficiently track, manage, and optimize your inventory. Dive in to learn how to streamline your processes and ensure seamless operations.

Purpose of API

In an Inventory Management System (IMS), authentication and authorization ensure that users access only the appropriate areas of the system based on their role. Authentication verifies a user's identity, while authorization controls what actions they can perform.

Installation

Welcome to the Inventory Management System! This comprehensive guide will walk you through the process of setting up and running the system locally on your machine using XAMPP.

Prerequisites

Before you begin, ensure you have the following installed on your device:

  • XAMPP (PHP and MySQL environment)
  • A modern web browser (e.g., Chrome, Firefox, Edge)

Installation Steps

Step 1: Clone the Repository

Open your terminal or command prompt. Clone the repository using the following command:

git clone https://github.com/Ivan-Lazz/imsfin

Once cloned, navigate to the project directory:

cd imsfin

Step 2: Move Folders to XAMPP's htdocs

Locate your XAMPP installation folder. The default path is typically:

  • Windows: C:\xampp\htdocs

Copy the following folder from the cloned repository into the htdocs directory:

  • imsfin

Step 3: Set Up the Database

Start XAMPP and ensure Apache and MySQL services are running. Then, open your browser and go to:

  • http://localhost/phpmyadmin

Create a new database:

  • Click on New in the left-side pane.
  • Name your database (e.g., php_ims) and click Create.

Import the SQL file:

  • Inside the main folder, locate the SQL file (e.g., php_ims.sql).
  • Go to the database you created in phpMyAdmin.
  • Click Import and select the corresponding SQL file.

Troubleshooting

If the website or Inventory Management doesn’t load, ensure that:

  • Apache and MySQL are running in XAMPP.
  • The folders are placed correctly in htdocs.
  • Database configurations (database.php) match your setup.

If you encounter a database error, check if you’ve imported the SQL files into the correct database.

API Overview

Our Inventory Management System (IMS) API is designed to simplify and optimize inventory management processes. Through a set of efficient RESTful endpoints, users can easily track stock levels, manage product details, and process orders in real-time. The API integrates seamlessly with other business systems, enabling automated updates and data access, which improves accuracy and minimizes manual intervention. Whether updating inventory, managing product catalogs, or generating reports, our IMS API offers a flexible, reliable solution for streamlining inventory workflows and enhancing operational efficiency.

Authentication API

The IMS provides an authentication API responsible for managing user login and session management, ensuring that only authorized users can access the system. It verifies user credentials against the database and initiates a session for successful logins, returning user data and a success message. The Authentication API plays a crucial role in maintaining the security and integrity of the web system by preventing unauthorized access.

Base URL

The base URL for the authentication API is:

http://localhost/imsfin/IMS_API/

Endpoints

The authentication API provides the following endpoints:

  • POST /api/auth/ : Authenticate user account.

Requests

Retrieve User Information

To login to the website, send a POST request to one of the following endpoints:

POST http://localhost/imsfin/IMS_API/api/auth/admin_auth.php
POST http://localhost/imsfin/IMS_API/api/auth/user_auth.php

The request body should contain the following parameters:

  • username: The username of the account.
  • password: The password of the account.

Example Request


POST /api/admin_auth.php
{
    "username": "admin",
    "password": "admin"
}

Example Response


{
    "status": 200,
    "message": "Login successful",
    "user": {
        "username": "admin",
        "firstname": "Koga",
        "lastname": "admin",
        "role": "admin"
    }
}

Errors

The following error codes may be returned in case of issues:

  • 400 Bad Request: Missing or invalid parameters.
  • 401 Unauthorized:Invalid or missing authentication token.
  • 403 Forbidden:Insufficient permissions.
  • 404 Page Not Found: Missing or deleted data.
  • 500 Server Error: Internal server error.

Example Error Response


{
    "status": 401,
    "message": "Invalid credentials"
}

Security and Performance

Security in a system involves protecting data, resources, and users from unauthorized access, breaches, or attacks through measures like encryption, authentication, and access controls. Performance refers to how efficiently the system operates, focusing on speed, responsiveness, and optimal resource usage, ensuring smooth functionality even under heavy demand.

Authentication & Authorization

In an Inventory Management System (IMS), authentication and authorization ensure that users access only the appropriate areas of the system based on their role. Authentication verifies a user's identity, while authorization controls what actions they can perform.

For Admins, authentication typically involves a username and password, with added security measures. Once logged in, admins are granted full access to all system functionalities, such as managing inventory, user roles, and system settings, based on their elevated privileges.

For Users, authentication also involves a secure login process, but their access is more restricted. Users can only view and interact with inventory data relevant to their role and permissions, ensuring they cannot modify critical settings or sensitive information in the system.



SECURITY

Session Management

The code uses PHP sessions to manage user authentication. For example, in files like add_new_company.php, it checks if the session variable $_SESSION['admin'] is set to determine if the user is logged in as an admin. If the session is not set, it redirects the user to the login page using JavaScript.

Input Validation and Sanitization

The code sanitizes user inputs using htmlspecialchars(strip_tags(...)) to prevent XSS (Cross-Site Scripting) attacks. This is seen in the authenticate.php file where the username and password are sanitized before being processed.

Prevent SQL Injection

By using a prepared statement with parameterized queries which separate SQL code from user input. This ensures that user-supplied values are treated as data rather than executable SQL, preventing malicious code injection.

Error Handling

The code includes error handling for various operations, such as creating, reading, updating, and deleting records. For instance, in create_company.php, it checks if the company name is provided and returns appropriate HTTP response codes and messages if the operation fails.

HTTP Response Code

The code uses appropriate HTTP response codes (e.g., 200 for success, 400 for bad requests, 401 for unauthorized access, 404 for not found, and 503 for service unavailable) to indicate the status of API requests.

Cross-Origin Resource Sharing

The code includes headers to allow cross-origin requests, which is important for APIs that may be accessed from different domains.


PERFORMANCE

Lightweight Framework

The application appears to be built without a heavy framework, which can lead to faster load times and reduced overhead. This lightweight approach can enhance performance, especially for simple CRUD operations.

Use of JSON for Data Exchange

The application uses JSON for data exchange between the server and client. JSON is lightweight and easy to parse, which can lead to faster data transmission and processing times compared to XML.

Asynchronous Data Fetching

Asynchronous fetching allows the application to continue running while waiting for data to be retrieved, meaning users can interact with other parts of the application without experiencing delays. It helps the overall loading of the page much faster.

Responsive Design

The use of responsive design principles ensures that the application performs well across various devices and screen sizes, enhancing user experience and engagement.

Minimal Dependencies

The IMS does not rely heavily on external libraries or frameworks, which can reduce load times and improve performance by minimizing the amount of code that needs to be processed.
